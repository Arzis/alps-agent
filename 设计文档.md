## 一、项目概述

### 1.1 项目定位

构建一个企业级智能问答助手平台，以 **RAG（检索增强生成）** 为核心能力，支持多智能体协作、工具调用、人工审查、可中断/恢复的对话流程，并提供高并发、可扩展的 FastAPI 服务接口。当 RAG 效果不佳时，系统自动降级至 Codex 作为兜底回答。

### 1.2 核心设计原则

| 原则            | 说明                                |
| --------------- | ----------------------------------- |
| **模块化**      | 各子系统松耦合，可独立迭代升级      |
| **可观测**      | 全链路追踪，LLM 调用可审计          |
| **可中断/恢复** | 任意节点可暂停、持久化、恢复        |
| **质量可控**    | 置信度评估 + 人工审查 + 自动降级    |
| **水平扩展**    | 无状态服务 + 分布式存储，支持高并发 |

### 1.3 补充的关键能力（基于经验建议）

在你列出的 11 项需求基础上，我建议补充以下能力：

| #    | 补充能力                              | 说明                                                    |
| ---- | ------------------------------------- | ------------------------------------------------------- |
| 12   | **查询理解与改写（Query Rewriting）** | 意图识别、多轮指代消解、查询扩展，直接影响 RAG 召回质量 |
| 13   | **答案置信度评估 & 自动降级**         | 对 RAG 回答进行打分，低于阈值时自动降级到 Codex         |

| 16   | **流式响应（SSE Streaming）**         | 提升用户体验，降低首 Token 延迟                         |

| 18   | **异步文档处理管道**                  | 文档上传后异步解析、分块、索引                          |
| 19   | **增量索引**                          | 文档更新时无需全量重建向量索引                          |
| 20   | **LLM 可观测性平台**                  | 集成 LangFuse/LangSmith 追踪每次 LLM 调用               |

| 22   | **多租户支持**                        | 企业级必备，不同租户数据隔离                            |

## 三、技术选型与对比

### 3.1 框架选型决策

| 框架           | 角色定位      | 是否采用         | 理由                                                         |
| -------------- | ------------- | ---------------- | ------------------------------------------------------------ |
| **LangGraph**  | 对话编排引擎  | ✅ **核心采用**   | 原生支持有状态图、Checkpoint持久化（可中断/恢复）、Human-in-the-loop、条件分支、多Agent编排，完美匹配需求5、6 |
| **LlamaIndex** | RAG 引擎      | ✅ **核心采用**   | 最佳的RAG原语支持：文档加载器、节点解析器、多种检索器、响应合成器，RAG能力远强于LangChain |
| **LangChain**  | 基础设施库    | ✅ **选择性采用** | 作为LangGraph的底层依赖，使用其Prompt模板、OutputParser、Tool抽象、MCP集成 |
| **FastAPI**    | API 服务框架  | ✅ **核心采用**   | 异步高性能、原生OpenAPI文档、类型提示、依赖注入              |
| **AutoGen**    | 多Agent对话   | ⏸️ **暂缓**       | LangGraph已覆盖多Agent场景，引入AutoGen增加架构复杂度        |
| **CrewAI**     | 角色Agent团队 | ⏸️ **暂缓**       | 同上，后续可作为特定场景的补充                               |

### 3.3 缓存方案深度对比

| 方案                 | 适用场景      | 推荐程度 | 说明                                                         |
| -------------------- | ------------- | -------- | ------------------------------------------------------------ |
| **Redis + 语义缓存** | 问答结果缓存  | ⭐⭐⭐⭐⭐    | Redis Stack 内置向量搜索，可实现语义相似度缓存，一套Redis解决多个问题 |
| **GPTCache**         | LLM调用缓存   | ⭐⭐⭐⭐     | 专为LLM设计的语义缓存库，支持多种相似度策略，但额外引入依赖  |
| **Redis (纯KV)**     | 会话/状态缓存 | ⭐⭐⭐⭐⭐    | 会话状态、LangGraph checkpoint、限流计数                     |
| **本地LRU**          | 热点数据      | ⭐⭐⭐      | 单实例热点数据缓存，配合Redis做二级缓存                      |

**推荐方案：Redis Stack 统一方案**

from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.postgres.aio import AsyncPostgresSaver
from langgraph.prebuilt import ToolNode

def build_conversation_graph() -> StateGraph:
    """构建对话编排图"""
    
    graph = StateGraph(ConversationState)
    
    # ============================================================
    # 注册节点
    # ============================================================
    graph.add_node("load_memory",        load_memory_node)
    graph.add_node("query_understanding", query_understanding_node)
    graph.add_node("intent_router",      intent_router_node)
    graph.add_node("rag_agent",          rag_agent_node)
    graph.add_node("tool_agent",         tool_agent_node)
    graph.add_node("chat_agent",         chat_agent_node)
    graph.add_node("multi_agent_hub",    multi_agent_hub_node)
    graph.add_node("quality_evaluator",  quality_evaluator_node)
    graph.add_node("codex_fallback",     codex_fallback_node)
    graph.add_node("human_review",       human_review_node)
    graph.add_node("response_synthesizer", response_synthesizer_node)
    graph.add_node("save_memory",        save_memory_node)
    
    # ============================================================
    # 定义边 (流程控制)
    # ============================================================
    
    # 入口
    graph.add_edge(START, "load_memory")
    graph.add_edge("load_memory", "query_understanding")
    graph.add_edge("query_understanding", "intent_router")
    
    # 意图路由 (条件分支)
    graph.add_conditional_edges(
        "intent_router",
        route_by_intent,
        {
            "knowledge_qa": "rag_agent",
            "tool_call":    "tool_agent",
            "chitchat":     "chat_agent",
            "complex_task": "multi_agent_hub",
            "unclear":      "response_synthesizer",  # 直接追问
        }
    )
    
    # 各Agent -> 质量评估
    graph.add_edge("rag_agent",        "quality_evaluator")
    graph.add_edge("tool_agent",       "quality_evaluator")
    graph.add_edge("chat_agent",       "response_synthesizer")  # 闲聊不需要质量评估
    graph.add_edge("multi_agent_hub",  "quality_evaluator")
    
    # 质量评估 (条件分支)
    graph.add_conditional_edges(
        "quality_evaluator",
        route_by_quality,
        {
            "pass":          "response_synthesizer",   # 质量合格
            "fallback_codex": "codex_fallback",        # 降级Codex
            "human_review":   "human_review",          # 人工审查
        }
    )
    
    # Codex降级后 -> 直接合成
    graph.add_edge("codex_fallback", "response_synthesizer")
    
    # 人工审查后 -> 合成 (这里会中断等待人工输入)
    graph.add_edge("human_review", "response_synthesizer")
    
    # 最终输出
    graph.add_edge("response_synthesizer", "save_memory")
    graph.add_edge("save_memory", END)
    
    return graph


def route_by_intent(state: ConversationState) -> str:
    """根据意图路由"""
    return state.intent


def route_by_quality(state: ConversationState) -> str:
    """根据质量评估结果路由"""
    if state.confidence_score >= 0.8:
        return "pass"
    elif state.confidence_score >= 0.5:
        return "fallback_codex"  
    else:
        return "human_review"


# ============================================================
# 编译图 (带持久化检查点)
# ============================================================
async def create_compiled_graph():
    """创建编译后的图，带PostgreSQL持久化"""
    
    checkpointer = AsyncPostgresSaver.from_conn_string(
        settings.POSTGRES_URL
    )
    await checkpointer.setup()  # 创建checkpoint表
    
    graph = build_conversation_graph()
    compiled = graph.compile(
        checkpointer=checkpointer,
        interrupt_before=["human_review"],  # 在人工审查前中断
    )
    return compiled

#### 4.2.3 可中断/恢复机制

Python



```
# src/core/orchestrator/checkpointer.py

class ConversationOrchestrator:
    """对话编排器 - 支持中断/恢复"""
    
    def __init__(self, compiled_graph, memory_manager, cache_manager):
        self.graph = compiled_graph
        self.memory = memory_manager
        self.cache = cache_manager
    
    async def run(
        self, 
        user_id: str, 
        session_id: str, 
        message: str,
        config: dict = None
    ) -> ConversationResult:
        """执行对话（可能在human_review节点中断）"""
        
        thread_config = {
            "configurable": {
                "thread_id": session_id,
                "user_id": user_id,
            }
        }
        
        initial_state = {
            "messages": [HumanMessage(content=message)],
            "user_id": user_id,
            "session_id": session_id,
            "original_query": message,
        }
        
        result = await self.graph.ainvoke(
            initial_state, 
            config=thread_config
        )
        
        # 检查是否被中断（等待人工审查）
        state = await self.graph.aget_state(thread_config)
        if state.next:  # 有待执行的节点 = 被中断了
            return ConversationResult(
                session_id=session_id,
                status="waiting_review",
                message="回答正在等待人工审查，请稍候...",
                interrupted_at=state.next[0],
            )
        
        return ConversationResult(
            session_id=session_id,
            status="completed",
            message=result["final_answer"],
            citations=result.get("citations", []),
            confidence=result.get("confidence_score", 0),
        )
    
    async def resume(
        self, 
        session_id: str, 
        human_review_result: HumanReviewResult,
        reviewer_id: str,
    ) -> ConversationResult:
        """恢复被中断的对话（人工审查完成后）"""
        
        thread_config = {
            "configurable": {"thread_id": session_id}
        }
        
        # 更新状态：注入人工审查结果
        await self.graph.aupdate_state(
            thread_config,
            values={
                "human_review_result": human_review_result,
                "needs_human_review": False,
            },
            as_node="human_review",  # 以human_review节点的身份更新
        )
        
        # 从中断点继续执行
        result = await self.graph.ainvoke(
            None,  # None表示从checkpoint恢复
            config=thread_config,
        )
        
        return ConversationResult(
            session_id=session_id,
            status="completed", 
            message=result["final_answer"],
            citations=result.get("citations", []),
        )
    
    async def stream(self, user_id, session_id, message):
        """流式执行，逐步返回事件"""
        thread_config = {
            "configurable": {"thread_id": session_id}
        }
        
        async for event in self.graph.astream_events(
            {"messages": [HumanMessage(content=message)], ...},
            config=thread_config,
            version="v2",
        ):
            if event["event"] == "on_chat_model_stream":
                yield StreamEvent(
                    type="token",
                    data=event["data"]["chunk"].content
                )
            elif event["event"] == "on_chain_end":
                if event["name"] == "rag_agent":
                    yield StreamEvent(
                        type="citations", 
                        data=event["data"]["output"]["citations"]
                    )
```

    2. 查询改写(提高检索质量)
    3. 查询扩展(多路召回)
    4. 意图预判
    """
    messages = state.messages
    original_query = state.original_query
    
    # 从记忆中获取上下文
    context_messages = messages[-10:]  # 最近5轮

        "rewritten_query": result.rewritten_query,

        "intent": result.intent,
        "latency_ms": {
            **state.latency_ms, 
            "query_understanding": timer.elapsed_ms

class MemoryManager:
    """统一记忆管理器"""
    
    def __init__(
        self,
        redis_client: Redis,
        pg_pool: asyncpg.Pool,
        milvus_client: MilvusClient,

        relevant_history = results[1] if not isinstance(results[1], Exception) else []
        user_entities = results[2] if len(results) > 2 and not isinstance(results[2], Exception) else []

class ShortTermMemory:
    """短期记忆 - Redis实现"""
    
    def __init__(self, redis: Redis):
        self.redis = redis
        self.ttl = 86400  # 24小时
    
    async def get_recent_messages(
        self, session_id: str, max_turns: int = 10
    ) -> list[ConversationTurn]:
        key = f"memory:short:{session_id}"
        raw = await self.redis.lrange(key, -max_turns * 2, -1)
        return [ConversationTurn.model_validate_json(r) for r in raw]
    
    async def append_turn(self, session_id: str, turn: ConversationTurn):
        key = f"memory:short:{session_id}"
        await self.redis.rpush(key, turn.model_dump_json())
        await self.redis.expire(key, self.ttl)
        
        # 如果超过阈值，触发摘要压缩
        length = await self.redis.llen(key)
        if length > 30:  # 超过15轮
            await self._compress_with_summary(session_id)
    
    async def _compress_with_summary(self, session_id: str):
        """用LLM对早期对话生成摘要，释放token空间"""
        key = f"memory:short:{session_id}"
        # 取前20条消息生成摘要
        old_messages = await self.redis.lrange(key, 0, 19)
        summary = await generate_summary(old_messages)
        # 删除旧消息，保留摘要 + 最近消息
        await self.redis.ltrim(key, 20, -1)
        summary_turn = ConversationTurn(
            user_message="[系统摘要]",
            assistant_message=summary,
            metadata={"is_summary": True}
        )
        await self.redis.lpush(key, summary_turn.model_dump_json())


class LongTermMemory:
    """长期记忆 - PostgreSQL + Milvus"""
    
    async def save_turn(
        self, user_id: str, session_id: str, turn: ConversationTurn
    ):
        # 1. 结构化存储到PostgreSQL
        await self.pg_pool.execute(
            """INSERT INTO conversation_history 
               (user_id, session_id, user_message, assistant_message, 
                metadata, created_at) 
               VALUES ($1, $2, $3, $4, $5, $6)""",
            user_id, session_id, turn.user_message,
            turn.assistant_message, json.dumps(turn.metadata),
            turn.timestamp,
        )
        
        # 2. 向量索引到Milvus（用于语义搜索历史）
        text_for_embedding = f"Q: {turn.user_message}\nA: {turn.assistant_message}"
        embedding = await self.embedding_model.aget_text_embedding(text_for_embedding)
        
        await self.milvus_client.insert(
            collection_name="conversation_memory",
            data=[{
                "user_id": user_id,
                "session_id": session_id,
                "text": text_for_embedding,
                "embedding": embedding,
                "timestamp": turn.timestamp.timestamp(),
            }],
        )
    
    async def search_relevant_history(
        self, user_id: str, query: str, top_k: int = 5
    ) -> list[HistoryResult]:
        """语义搜索用户历史对话"""
        query_embedding = await self.embedding_model.aget_text_embedding(query)
        
        results = await self.milvus_client.search(
            collection_name="conversation_memory",
            data=[query_embedding],
            filter=f'user_id == "{user_id}"',
            limit=top_k,
            output_fields=["text", "session_id", "timestamp"],
        )
        
        return [
            HistoryResult(
                text=hit["entity"]["text"],
                score=hit["distance"],
                session_id=hit["entity"]["session_id"],
            )
            for hit in results[0]
        ]

from llama_index.core.node_parser import (

from llama_index.readers.file import (

class DocumentIngestionPipeline:
    """文档摄取管道 - 解析 -> 分块 -> 元数据提取 -> Embedding -> 索引"""
    
    def __init__(self, config: RAGConfig):
        self.config = config
        self.parsers = self._init_parsers()
        self.chunkers = self._init_chunkers()
        self.embedding_model = self._init_embedding()
        self.milvus_store = self._init_vector_store()
        self.es_store = self._init_doc_store()
    
    def _init_parsers(self) -> dict:
        """初始化文档解析器"""
        return {
            ".pdf": PDFReader(return_full_document=False),
            ".docx": DocxReader(),
            ".md": MarkdownReader(),
            ".xlsx": PandasExcelReader(),
            ".txt": SimpleDirectoryReader,
            # 高级: 使用Unstructured.io处理复杂文档
            # ".pdf_complex": UnstructuredReader(strategy="hi_res"),
        }
    
    def _init_chunkers(self) -> dict:
        """初始化分块策略"""
        return {
            # 策略1: 递归字符分割 (通用、稳定)
            "recursive": SentenceSplitter(
                chunk_size=512,
                chunk_overlap=50,
                paragraph_separator="\n\n",
            ),
            
            # 策略2: 语义分割 (基于Embedding相似度自适应分块)
            "semantic": SemanticSplitterNodeParser(
                buffer_size=1,
                breakpoint_percentile_threshold=95,
                embed_model=self.embedding_model,
            ),
            
            # 策略3: 层次分割 (用于Auto-Merging Retrieval)
            "hierarchical": HierarchicalNodeParser.from_defaults(
                chunk_sizes=[2048, 512, 128],  # 大/中/小三级
            ),
            
            # 策略4: Agentic Chunking (LLM判断分块边界，质量最高但成本高)
            # "agentic": AgenticChunker(llm=self.llm),
        }
    
    async def process(
        self, 
        doc_id: str, 
        collection: str,
        chunking_strategy: str = "semantic",  # 默认语义分块
    ):
        """完整的文档处理流程"""
        
        try:
            # 1. 加载原始文档
            raw_doc = await self._load_raw_document(doc_id)
            
            # 2. 解析文档
            documents = self._parse_document(raw_doc)
            
            # 3. 分块
            chunker = self.chunkers[chunking_strategy]
            nodes = chunker.get_nodes_from_documents(documents)
            
            # 4. 元数据增强
            nodes = await self._enrich_metadata(nodes, raw_doc)
            
            # 5. 构建摄取管道
            pipeline = IngestionPipeline(
                transformations=[
                    # 元数据提取
                    TitleExtractor(llm=self.llm),
                    KeywordExtractor(llm=self.llm, keywords=5),
                    QuestionsAnsweredExtractor(llm=self.llm, questions=3),
                    SummaryExtractor(llm=self.llm),
                    # Embedding
                    self.embedding_model,
                ],
                vector_store=self.milvus_store,
                docstore=self.es_store,  # 存储原始节点到ES(用于BM25)
            )
            
            # 6. 执行管道
            processed_nodes = await pipeline.arun(
                nodes=nodes,
                show_progress=True,
                num_workers=4,  # 并行处理
            )
            
            # 7. 更新文档状态
            await self._update_doc_status(doc_id, "completed", len(processed_nodes))
            
            logger.info(
                "document_ingested",
                doc_id=doc_id,
                collection=collection,
                num_chunks=len(processed_nodes),
                strategy=chunking_strategy,
            )
            
        except Exception as e:
            await self._update_doc_status(doc_id, "failed", error=str(e))
            logger.error("document_ingestion_failed", doc_id=doc_id, error=str(e))
            raise
    
    async def _enrich_metadata(self, nodes, raw_doc):
        """元数据增强"""
        for node in nodes:
            node.metadata.update({
                "doc_id": raw_doc.id,
                "doc_title": raw_doc.title,
                "source_type": raw_doc.file_type,
                "upload_time": raw_doc.created_at.isoformat(),
                "collection": raw_doc.collection,
                # 用于Milvus分区
                "partition_key": raw_doc.collection,  
            })
        return nodes

    async def incremental_update(self, doc_id: str, collection: str):
        """增量更新 - 只更新变化的文档"""
        # 1. 删除旧的向量
        await self.milvus_store.delete(
            filter=f'doc_id == "{doc_id}"'
        )
        # 2. 删除旧的ES文档
        await self.es_store.delete_document(doc_id)
        # 3. 重新处理
        await self.process(doc_id, collection)

# src/core/rag/retrieval/hybrid_retriever.py

from llama_index.core.retrievers import BaseRetriever
from llama_index.core.schema import NodeWithScore

class HybridRetriever(BaseRetriever):
    """
    多路混合检索器
    1. Dense Retrieval  - Milvus向量检索
    2. Sparse Retrieval - Elasticsearch BM25
    3. KG Retrieval     - Neo4j知识图谱 (可选)
    
    融合策略: Reciprocal Rank Fusion (RRF)
    """
    
    def __init__(
        self,
        dense_retriever: MilvusDenseRetriever,
        sparse_retriever: ElasticsearchSparseRetriever,
        kg_retriever: KGRetriever | None = None,
        reranker: BaseReranker = None,
        dense_weight: float = 0.5,
        sparse_weight: float = 0.3,
        kg_weight: float = 0.2,
        top_k: int = 10,
        rerank_top_n: int = 5,
    ):
        self.dense = dense_retriever
        self.sparse = sparse_retriever
        self.kg = kg_retriever
        self.reranker = reranker
        self.weights = {
            "dense": dense_weight,
            "sparse": sparse_weight,
            "kg": kg_weight,
        }
        self.top_k = top_k
        self.rerank_top_n = rerank_top_n
    
    async def _aretrieve(self, query_bundle) -> list[NodeWithScore]:
        """异步多路召回 + 融合 + 重排"""
        
        query = query_bundle.query_str
        
        # ============================================================
        # Step 1: 并行多路召回
        # ============================================================
        retrieval_tasks = [
            self.dense.aretrieve(query, top_k=self.top_k * 2),
            self.sparse.aretrieve(query, top_k=self.top_k * 2),
        ]
        if self.kg:
            retrieval_tasks.append(
                self.kg.aretrieve(query, top_k=self.top_k)
            )
        
        results = await asyncio.gather(*retrieval_tasks, return_exceptions=True)
        
        dense_results = results[0] if not isinstance(results[0], Exception) else []
        sparse_results = results[1] if not isinstance(results[1], Exception) else []
        kg_results = results[2] if len(results) > 2 and not isinstance(results[2], Exception) else []
        
        # 标记来源
        for node in dense_results:
            node.metadata["retrieval_source"] = "dense"
        for node in sparse_results:
            node.metadata["retrieval_source"] = "sparse"
        for node in kg_results:
            node.metadata["retrieval_source"] = "kg"
        
        # ============================================================
        # Step 2: Reciprocal Rank Fusion (RRF)
        # ============================================================
        fused_results = self._reciprocal_rank_fusion(
            result_lists={
                "dense": dense_results,
                "sparse": sparse_results,
                "kg": kg_results,
            },
            k=60,  # RRF常量
        )
        
        # 取Top-K
        fused_top_k = fused_results[:self.top_k]
        
        # ============================================================
        # Step 3: Rerank (精排)
        # ============================================================
        if self.reranker and fused_top_k:
            reranked = await self.reranker.arerank(
                query=query,
                nodes=fused_top_k,
                top_n=self.rerank_top_n,
            )
            return reranked
        
        return fused_top_k[:self.rerank_top_n]
    
    def _reciprocal_rank_fusion(
        self, 
        result_lists: dict[str, list[NodeWithScore]], 
        k: int = 60
    ) -> list[NodeWithScore]:
        """
        RRF融合算法
        score(d) = Σ (weight_i / (k + rank_i(d)))
        """
        doc_scores: dict[str, float] = {}
        doc_nodes: dict[str, NodeWithScore] = {}
        
        for source, results in result_lists.items():
            weight = self.weights.get(source, 0.0)
            for rank, node in enumerate(results, start=1):
                node_id = node.node.node_id
                rrf_score = weight / (k + rank)
                doc_scores[node_id] = doc_scores.get(node_id, 0.0) + rrf_score
                if node_id not in doc_nodes:
                    doc_nodes[node_id] = node
        
        # 按RRF分数排序
        sorted_ids = sorted(doc_scores.keys(), key=lambda x: doc_scores[x], reverse=True)
        
        return [
            NodeWithScore(
                node=doc_nodes[nid].node,
                score=doc_scores[nid],
            )
            for nid in sorted_ids
        ]


# src/core/rag/retrieval/reranker.py

class CrossEncoderReranker:
    """Cross-Encoder 重排器"""
    
    def __init__(self, model_name: str = "BAAI/bge-reranker-v2-m3"):
        from sentence_transformers import CrossEncoder
        self.model = CrossEncoder(model_name, max_length=512)
    
    async def arerank(
        self, 
        query: str, 
        nodes: list[NodeWithScore], 
        top_n: int = 5
    ) -> list[NodeWithScore]:
        """使用Cross-Encoder精排"""
        
        pairs = [(query, node.node.get_content()) for node in nodes]
        
        # Cross-Encoder推理（CPU/GPU密集，放到线程池）
        scores = await asyncio.get_event_loop().run_in_executor(
            None,
            lambda: self.model.predict(pairs)
        )
        
        # 按Rerank分数排序
        scored_nodes = [
            NodeWithScore(node=node.node, score=float(score))
            for node, score in zip(nodes, scores)
        ]
        scored_nodes.sort(key=lambda x: x.score, reverse=True)
        
        return scored_nodes[:top_n]

            field_name="tenant_id",
            index_type="Trie",

            search_params={
                "metric_type": "COSINE",
                "params": {"ef": 128},  # HNSW搜索参数

from langgraph.graph import StateGraph, START, END
from langgraph.types import Command, interrupt
from langchain_core.messages import HumanMessage, SystemMessage

class MultiAgentState(TypedDict):
    """多智能体状态"""
    messages: Annotated[list, add_messages]
    task: str                              # 当前任务描述
    subtasks: list[SubTask]                # 分解后的子任务
    agent_outputs: dict[str, AgentOutput]  # 各Agent输出
    debate_rounds: list[DebateRound]       # 辩论记录
    final_consensus: str                   # 最终共识
    iteration: int                         # 当前迭代轮次
    max_iterations: int                    # 最大迭代次数


def build_multi_agent_graph() -> StateGraph:
    """构建多智能体协作/竞争图"""
    
    graph = StateGraph(MultiAgentState)
    
    # === 节点 ===
    graph.add_node("supervisor",     supervisor_node)
    graph.add_node("rag_expert",     rag_expert_node)
    graph.add_node("tool_expert",    tool_expert_node)
    graph.add_node("analyst",        analyst_node)
    graph.add_node("critic",         critic_node)
    graph.add_node("debate_arena",   debate_arena_node)
    graph.add_node("judge",          judge_node)
    graph.add_node("synthesizer",    synthesizer_node)
    
    # === 边 ===
    graph.add_edge(START, "supervisor")
    
    # Supervisor 动态路由到不同Agent
    graph.add_conditional_edges(
        "supervisor",
        supervisor_route,
        {
            "rag_expert":    "rag_expert",
            "tool_expert":   "tool_expert",
            "analyst":       "analyst",
            "debate":        "debate_arena",
            "synthesize":    "synthesizer",
            "done":          END,
        }
    )
    
    # 各Expert完成后回到Supervisor
    graph.add_edge("rag_expert",  "supervisor")
    graph.add_edge("tool_expert", "supervisor")
    graph.add_edge("analyst",     "supervisor")
    
    # 辩论流程
    graph.add_edge("debate_arena", "critic")
    graph.add_conditional_edges(
        "critic",
        should_continue_debate,
        {
            "continue": "debate_arena",  # 继续辩论
            "judge":    "judge",          # 进入裁决
        }
    )
    graph.add_edge("judge", "supervisor")
    
    # 最终合成
    graph.add_edge("synthesizer", END)
    
    return graph


async def supervisor_node(state: MultiAgentState) -> Command:
    """
    Supervisor Agent - 智能调度中心
    分析当前状态，决定下一步调用哪个Agent
    """
    
    supervisor_prompt = """你是一个智能任务调度器。根据当前任务状态，决定下一步行动。

当前任务: {task}
已完成的Agent输出: {agent_outputs}
当前迭代: {iteration}/{max_iterations}

你可以选择:
1. "rag_expert" - 需要从知识库检索信息
2. "tool_expert" - 需要调用工具或API
3. "analyst" - 需要深度分析或推理
4. "debate" - 答案存在不确定性，需要辩论机制提高质量
5. "synthesize" - 所有信息已足够，可以合成最终答案
6. "done" - 任务已完成

请返回你的决策和理由。"""
    
    response = await llm.ainvoke([
        SystemMessage(content=supervisor_prompt.format(
            task=state["task"],
            agent_outputs=json.dumps(state["agent_outputs"], ensure_ascii=False),
            iteration=state["iteration"],
            max_iterations=state["max_iterations"],
        ))
    ])
    
    decision = parse_supervisor_decision(response)
    
    # 防止无限循环
    if state["iteration"] >= state["max_iterations"]:
        decision.next_agent = "synthesize"
    
    return Command(
        goto=decision.next_agent,
        update={"iteration": state["iteration"] + 1},
    )


async def debate_arena_node(state: MultiAgentState) -> dict:
    """
    辩论场 - 多Agent竞争机制
    正方和反方就同一问题给出不同观点
    """
    
    question = state["task"]
    current_context = state["agent_outputs"]
    previous_rounds = state["debate_rounds"]
    
    # 正方: 支持当前最佳答案
    proponent_response = await llm.ainvoke([
        SystemMessage(content="""你是正方辩手。你需要:
1. 支持并强化当前答案
2. 引用证据证明答案的正确性
3. 反驳之前反方提出的质疑"""),
        HumanMessage(content=f"问题: {question}\n当前答案: {current_context}\n前几轮辩论: {previous_rounds}")
    ])
    
    # 反方: 质疑当前答案
    opponent_response = await llm.ainvoke([
        SystemMessage(content="""你是反方辩手。你需要:
1. 批判性地审视当前答案
2. 找出可能的错误、遗漏或逻辑漏洞
3. 提出更好的替代答案或补充"""),
        HumanMessage(content=f"问题: {question}\n当前答案: {current_context}\n正方论点: {proponent_response.content}")
    ])
    
    new_round = DebateRound(
        round_number=len(previous_rounds) + 1,
        proponent_argument=proponent_response.content,
        opponent_argument=opponent_response.content,
    )
    
    return {
        "debate_rounds": [*previous_rounds, new_round],
    }


async def judge_node(state: MultiAgentState) -> dict:
    """
    裁判 - 评估辩论结果，得出最佳答案
    """
    
    judge_response = await llm.ainvoke([
        SystemMessage(content="""你是公正的裁判。阅读所有辩论轮次后:
1. 评估正反双方的论点强度
2. 判断哪些观点有充分的证据支持
3. 综合得出最准确、最完整的答案
4. 标注答案的置信度 (0-1)"""),
        HumanMessage(content=f"辩论记录: {json.dumps(state['debate_rounds'], ensure_ascii=False)}")
    ])
    
    result = parse_judge_result(judge_response)
    
    return {
        "agent_outputs": {
            **state["agent_outputs"],
            "debate_result": {
                "answer": result.answer,
                "confidence": result.confidence,
                "reasoning": result.reasoning,
            }
        }
    }


def should_continue_debate(state: MultiAgentState) -> str:
    """判断是否继续辩论"""
    rounds = state["debate_rounds"]
    
    # 最多3轮辩论
    if len(rounds) >= 3:
        return "judge"
    
    # 如果正反方已达成共识
    if rounds and is_consensus_reached(rounds[-1]):
        return "judge"
    
    return "continue"

class RAGCompetitionStrategy:
    """
    多路RAG竞赛策略
    同一问题用不同RAG策略生成答案，选择最优
    """
    
    async def compete(self, query: str, context: dict) -> CompetitionResult:
        """并行执行多种RAG策略，竞争产出最佳答案"""
        
        strategies = [
            # 策略1: 标准RAG (Dense检索 + GPT-4o)
            self._strategy_standard_rag(query),
            # 策略2: HyDE (假设文档嵌入)
            self._strategy_hyde_rag(query),
            # 策略3: Step-back Prompting (抽象后再检索)
            self._strategy_stepback_rag(query),
            # 策略4: 多查询扩展 (MultiQuery)
            self._strategy_multiquery_rag(query),
        ]
        
        # 并行执行所有策略
        results = await asyncio.gather(*strategies, return_exceptions=True)
        
        # 过滤失败的
        valid_results = [
            r for r in results 
            if not isinstance(r, Exception) and r is not None
        ]
        
        if not valid_results:
            return CompetitionResult(winner=None, fallback=True)
        
        # Judge评分
        winner = await self._judge_best_answer(query, valid_results)
        
        return winner
    
    async def _judge_best_answer(
        self, query: str, candidates: list[StrategyResult]
    ) -> CompetitionResult:
        """Judge Agent 评估所有候选答案"""
        
        judge_prompt = f"""你是答案质量评估专家。以下是同一问题的多个候选答案:

问题: {query}

{self._format_candidates(candidates)}

请从以下维度评估每个答案 (1-10分):
1. 准确性 (Accuracy): 答案是否正确
2. 完整性 (Completeness): 是否覆盖了问题的所有方面
3. 相关性 (Relevancy): 是否切题
4. 引用质量 (Citation): 引用的来源是否可靠
5. 可读性 (Readability): 表述是否清晰

输出JSON格式的评分结果，并选出最佳答案。"""
        
        result = await call_llm_structured(judge_prompt, JudgeOutput)
        
        best_idx = result.best_candidate_index
        return CompetitionResult(
            winner=candidates[best_idx],
            scores=result.scores,
            reasoning=result.reasoning,

                NumericField("$.timestamp", as_name="timestamp"),

                ),
            )
            
            await self.redis.ft(self.index_name).create_index(
                schema,
                definition=IndexDefinition(
                    prefix=["cache:semantic:"],
                    index_type=IndexType.JSON,
                ),
            )
    
    async def get(self, query: str) -> CacheResult | None:
        """查询语义缓存"""
        
        # 1. 计算query embedding
        query_embedding = await self.embedding.aget_text_embedding(query)
        embedding_bytes = np.array(query_embedding, dtype=np.float32).tobytes()
        
        # 2. Redis向量搜索
        redis_query = (
            Query(f"*=>[KNN 3 @embedding $vec AS score]")
            .sort_by("score")
            .return_fields("query", "answer", "citations", "confidence", "score")
            .dialect(2)
        )
        
        results = await self.redis.ft(self.index_name).search(
            redis_query,
            query_params={"vec": embedding_bytes},
        )
        
        if not results.docs:
            return None
        
        # 3. 检查相似度阈值
        best = results.docs[0]
        similarity = 1 - float(best.score)  # COSINE distance -> similarity
        
        if similarity >= self.threshold:
            logger.info(
                "semantic_cache_hit",
                query=query,
                cached_query=best.query,
                similarity=similarity,
            )
            return CacheResult(
                answer=best.answer,
                citations=json.loads(best.citations),
                confidence=float(best.confidence),
                cached=True,
                similarity=similarity,
            )
        
        return None
    
    async def set(
        self, 
        query: str, 
        answer: str, 
        citations: list = None,
        confidence: float = 0.0,
    ):
        """写入语义缓存"""
        
        query_embedding = await self.embedding.aget_text_embedding(query)
        
        cache_key = f"cache:semantic:{hashlib.md5(query.encode()).hexdigest()}"
        
        await self.redis.json().set(cache_key, "$", {
            "query": query,
            "answer": answer,
            "citations": json.dumps(citations or []),
            "confidence": confidence,
            "embedding": query_embedding,
            "timestamp": int(datetime.utcnow().timestamp()),
        })
        
        await self.redis.expire(cache_key, self.ttl)


class CacheManager:
    """统一缓存管理器"""
    
    def __init__(self, redis_client, embedding_model):
        self.semantic_cache = SemanticCache(redis_client, embedding_model)
        self.redis = redis_client
    
    async def get_or_compute(
        self, 
        query: str, 
        compute_fn,
        cache_level: str = "semantic",
    ):
        """缓存查询，未命中则计算并缓存"""
        
        # Level 1: 精确匹配 (最快)
        exact_key = f"cache:exact:{hashlib.md5(query.encode()).hexdigest()}"
        cached = await self.redis.get(exact_key)
        if cached:
            return CacheResult.from_json(cached, cache_type="exact")
        
        # Level 2: 语义匹配
        if cache_level == "semantic":
            cached = await self.semantic_cache.get(query)
            if cached:
                return cached
        
        # 未命中，执行计算
        result = await compute_fn(query)
        
        # 写入缓存
        await self.redis.set(
            exact_key, 
            result.to_json(), 
            ex=3600
        )
        if cache_level == "semantic" and result.confidence > 0.7:
            await self.semantic_cache.set(
                query, 
                result.answer, 
                result.citations,
                result.confidence,
            )
        
        return result

class RAGEvaluator:
    """RAG质量评估器"""
    
    def __init__(self, llm, embedding_model):
        self.llm = llm
        self.embedding = embedding_model
        self.metrics = [
            faithfulness,        # 忠实度: 答案是否基于context
            answer_relevancy,    # 答案相关性
            context_precision,   # 上下文精确度
            context_recall,      # 上下文召回率
            answer_correctness,  # 答案正确性 (需要参考答案)
        ]
    
    async def evaluate_single(
        self, 
        query: str, 
        answer: str, 
        contexts: list[str],
        reference: str = None,
    ) -> EvaluationResult:
        """评估单条问答"""
        
        data = {
            "question": [query],
            "answer": [answer],
            "contexts": [contexts],
        }
        if reference:
            data["ground_truth"] = [reference]
        
        dataset = Dataset.from_dict(data)
        
        results = evaluate(
            dataset=dataset,
            metrics=self.metrics,
            llm=self.llm,
            embeddings=self.embedding,
        )
        
        return EvaluationResult(
            faithfulness=results["faithfulness"],
            answer_relevancy=results["answer_relevancy"],
            context_precision=results["context_precision"],
            context_recall=results.get("context_recall", None),
        )
    
    async def evaluate_batch(
        self, 
        test_dataset: list[TestCase],
    ) -> BatchEvaluationReport:
        """批量评估 (用于定期质量检查)"""
        
        data = {
            "question": [t.query for t in test_dataset],
            "answer": [t.answer for t in test_dataset],
            "contexts": [t.contexts for t in test_dataset],
            "ground_truth": [t.reference for t in test_dataset],
        }
        
        dataset = Dataset.from_dict(data)
        results = evaluate(
            dataset=dataset,
            metrics=self.metrics,
            llm=self.llm,
            embeddings=self.embedding,
        )
        
        return BatchEvaluationReport(
            total_samples=len(test_dataset),
            avg_faithfulness=results["faithfulness"].mean(),
            avg_relevancy=results["answer_relevancy"].mean(),
            avg_context_precision=results["context_precision"].mean(),
            detailed_results=results.to_pandas(),
            timestamp=datetime.utcnow(),
        )
    
    async def generate_test_set(
        self,
        documents: list,
        test_size: int = 50,
    ) -> list[TestCase]:
        """自动生成测试集 (基于RAGAS TestsetGenerator)"""
        
        generator = TestsetGenerator.from_langchain(
            generator_llm=self.llm,
            critic_llm=self.llm,
            embeddings=self.embedding,
        )
        
        testset = generator.generate_with_llamaindex_docs(
            documents=documents,
            test_size=test_size,
            distributions={
                "simple": 0.4,
                "reasoning": 0.3,
                "multi_context": 0.3,
            },
        )
        
        return testset

class SafetyGuardrails:
    """输入/输出安全护栏"""
    
    async def check_input(self, query: str) -> GuardrailResult:
        """输入安全检查"""
        checks = await asyncio.gather(
            self._check_prompt_injection(query),
            self._check_sensitive_info(query),
            self._check_topic_restriction(query),
        )
        
        for check in checks:
            if not check.passed:
                return check
        
        return GuardrailResult(passed=True)
    
    async def check_output(
        self, answer: str, context: list[str]
    ) -> GuardrailResult:
        """输出安全检查"""
        checks = await asyncio.gather(
            self._check_hallucination(answer, context),
            self._check_sensitive_output(answer),
            self._check_bias(answer),
        )
        
        for check in checks:
            if not check.passed:
                return check
        
        return GuardrailResult(passed=True)
    
    async def _check_prompt_injection(self, query: str) -> GuardrailResult:
        """检测Prompt注入攻击"""
        injection_patterns = [
            r"ignore\s+(all\s+)?previous\s+instructions",
            r"system\s*prompt",
            r"you\s+are\s+now",
            # ... 更多模式
        ]
        for pattern in injection_patterns:
            if re.search(pattern, query, re.IGNORECASE):
                return GuardrailResult(
                    passed=False,
                    reason="检测到可能的Prompt注入攻击",
                    action="block",
                )
        return GuardrailResult(passed=True)

## 七、关键设计决策摘要

| 决策点      | 选择               | 核心理由                                                     |
| ----------- | ------------------ | ------------------------------------------------------------ |
| 编排引擎    | LangGraph          | 原生Checkpoint(可中断/恢复)、Human-in-the-loop、状态机，是唯一满足需求5+6的框架 |
| RAG引擎     | LlamaIndex         | RAG原语最丰富，远优于LangChain的RAG能力                      |
| 向量库      | Milvus分布式       | 原生分布式、分区键(多租户)、HNSW高精度                       |
| 缓存        | Redis Stack        | 一套解决KV缓存+语义缓存+消息队列+分布式锁                    |
| 分块策略    | 语义分块(默认)     | 自适应边界，比固定长度分块质量高30%+                         |
| 重排        | bge-reranker-v2-m3 | 开源最强多语言Reranker，无API成本                            |
| LLM追踪     | LangFuse(自部署)   | 开源、数据不出域、企业友好                                   |
| 日志        | Structlog + Loki   | 结构化日志+轻量级聚合，比ELK节省资源60%                      |
| 多Agent竞争 | LangGraph辩论机制  | 正/反方辩论+Judge裁决，提升答案可靠性                        |
| 异步任务    | ARQ (Redis-based)  | 比Celery轻量10x，原生async，共用Redis                        |
